Day 15

Authentication Theory

Mastering identity. Learn how to securely recognize users in a stateless web environment.

1 The Hashing Gold Standard

Passwords must never be stored in plain text or encrypted. They must be hashed. Hashing is a one-way cryptographic function that protects user data even if your database is stolen.

Hashing (One-Way)

Input "123" becomes "$2b$12$Kjs...". There is no way to turn that hash back into "123".

Encryption (Two-Way)

Can be reversed with a key. Never use this for passwords because keys can be leaked.

2 The Authentication Flow

User submits Password
Server hashes input + Salt
Compare against stored Hash

3 Sessions vs. Tokens

Feature Sessions (Cookies) Tokens (JWT)
Storage Server-side Memory/DB Stateless (Client-only)
Scaling Harder (Needs shared DB) Easier (No DB lookup)
Best Use Standard Web Apps Mobile & APIs

The Fatal Five Mistakes

  • Storing any secret (like a real password) in a cookie.
  • Rolling your own hashing algorithm. (Use Bcrypt or Argon2).
  • Forgetting to Salt your hashes.
  • Mixing Auth (Identity) with Authorization (Permissions).
