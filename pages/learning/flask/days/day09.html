Day 9

Database Theory

The blueprint of the backend. Understanding how data survives, connects, and stays consistent.

1 Memory vs. Persistence

RAM (Memory)

  • • Fast, but temporary
  • • Lost on server restart
  • • Where Python variables live

Database (Disk)

  • • Persistent and shared
  • • Survives restarts & crashes
  • • Structured for fast access

3 Relationships & Keys

Primary Key (PK)

The unique passport for every row. It must never change and never repeat.

Foreign Key (FK)

A column that points to the Primary Key of another table. This is how we link a Post to its Author.

4 The CRUD Operations

Operation Database Action
Create Adding a new row (Signup, New Post)
Read Fetching data (Profile page, Feed)
Update Modifying a row (Edit Profile, Change Password)
Delete Removing data (Delete Account, Remove Comment)

Day 9 Outcome

Before writing code, you must master the "Think before you store" mindset. You should now be able to:

  • Differentiate RAM vs. Disk storage
  • Map real-world items to Tables/Rows
  • Explain why PKs are mandatory
  • Understand SQL vs. NoSQL basics
