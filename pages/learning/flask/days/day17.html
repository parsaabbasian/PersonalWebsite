Day 17

Advanced Channels in Go

Mastering buffered channels, complex synchronization with select, and advanced concurrency patterns to prevent deadlocks.

1 Buffered Channels

Buffered channels allow sending a limited number of values without a corresponding receiver. This decouples the sender and receiver until the buffer is full.

Capacity & Blocking

ch := make(chan int, 2) // Buffer of 2

ch <- 1 // Doesn't block
ch <- 2 // Doesn't block
ch <- 3 // BLOCKS (Buffer full)

2 The select Statement

The select statement lets a goroutine wait on multiple communication operations. It blocks until one of its cases can run.

Multiplexing

select {
case msg1 := <-ch1:
    fmt.Println(msg1)
case msg2 := <-ch2:
    fmt.Println(msg2)
}

Timeouts

select {
case msg := <-ch:
    fmt.Println(msg)
case <-time.After(time.Second):
    fmt.Println("Timeout")
}

3 Fixing the Loop Capture Bug

Passing the loop variable as an argument to the anonymous function ensures each goroutine gets a local copy of the value at that iteration.

for i := 0; i < 5; i++ {
    go func(n int) {
        fmt.Println(n)
    }(i) // Pass i here
}
