Mini Project 1: Putting it Together

The transition from learning concepts to building systems. Plan, design, and execute your first real-world Flask application.

1 The Backend Developer Workflow

Step 1: Define Entities

Decide what data objects exist (e.g., Users, Notes, Tasks).

Step 2: Schema Design

Design tables and relationships (Day 9 theory).

Step 3: Route Mapping

Plan your URLs and HTTP methods for each action.

Think in Features

"Create a Note" is a feature. "Insert row into DB" is the technical implementation. Always start with the feature.

2 The Data Blueprint

Before coding, sketch your relationship. This prevents circular imports and migration nightmares later.

User Model

  • id (PK)
  • username (Unique)
  • password (Hashed)

Note Model

  • id (PK)
  • title
  • content
  • user_id (FK)

3 Mapping Features to CRUD

Feature URL CRUD / HTTP
Add Note /notes/new Create (POST)
Browse Notes /notes Read (GET)
Edit Note /notes/<id>/edit Update (POST)
Remove Note /notes/<id>/delete Delete (POST)

The Architecture Audit

Even for a mini-project, follow professional standards. Does your app use:

  • Blueprints

    Separate routes for 'auth' and 'notes' into modules.

  • App Factory

    Using create_app() to prevent circular imports.

  • CSRF & Validation

    Protecting every POST request with Flask-WTF tokens.
