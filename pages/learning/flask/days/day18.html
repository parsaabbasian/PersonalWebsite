Day 18

sync.WaitGroup & Race Conditions

Moving beyond time.Sleep to professional synchronization and identifying data safety issues.

1 Professional Synchronization

sync.WaitGroup is the correct way to wait for a collection of goroutines to finish. It tracks a counter that increments when tasks start and decrements when they finish.

waitgroup.go
func worker(id int, wg *sync.WaitGroup) {
    defer wg.Done() // Decrement counter when done
    fmt.Println("Worker", id, "done")
}

func main() {
    var wg sync.WaitGroup
    for i := 1; i <= 3; i++ {
        wg.Add(1) // Increment before starting goroutine
        go worker(i, &wg)
    }
    wg.Wait() // Block until counter is 0
}

2 Race Conditions

The Danger of Shared Memory

A race condition occurs when multiple goroutines access the same variable simultaneously and at least one access is a write. This results in unpredictable behavior.

// ‚ùå DANGEROUS CODE
count := 0
go func() { count++ }()
go func() { count++ }()

go run -race main.go

Use Go's built-in detector!

3 WaitGroup vs. Channels

Use Case Tool
Waiting for goroutines to finish WaitGroup
Transferring data between goroutines Channels
Orchestrating complex async flows Both (Combined)
