Day 3

Flask Internals & Lifecycle

Understanding how Flask lives, runs, and handles thousands of users without mixing up their data.

1 The Anatomy of Flask(__name__)

Flask needs to know where your files are. By passing __name__, you give Flask a reference point to find your templates and static assets.

from flask import Flask

app = Flask(__name__) // Tells Flask: "Look for files in this folder"

2 The Application Lifecycle

It is critical to distinguish between things that happen once and things that happen per request.

Startup Time (Once)

  • • App Object Created
  • • Config & Env Loaded
  • • Routes Registered

Request Time (Per User)

  • • Context Created
  • • View Function Runs
  • • Context Destroyed

3 Isolation via Contexts

Flask uses "Contexts" to ensure that User A's data never leaks into User B's request. This is why Flask is Thread-Safe.

Application Context

Shared across the whole app. Contains configuration and DB connection settings.

Request Context

Specific to ONE user. Contains headers, form data, and the current session.

⚠️ Backend Rules of Thumb

X

Don't store data in Globals. If you set a global variable for one user, every other user will see it. Use the session object instead.

X

Avoid top-level logic. Code outside of route functions runs exactly once (at startup). It cannot access user data like request.args.

✓

Debug Mode is for Dev only. It provides a stack trace that reveals your source code to hackers. Turn it off in production!
