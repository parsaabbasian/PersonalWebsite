Day 7

Static Files & Project Structure

Mastering asset management and professional software organization to build scalable applications.

1 What are Static Files?

Static files are assets that do not change per user and are served "as-is" by the server. Unlike templates, the backend performs zero processing on these files.

CSS (Styles)
JS (Logic)
Images/Icons

Key Rule: Assets vs Templates

Templates are dynamic (rendered per request). Static files are fixed and cached by browsers to improve speed and reduce server load.

2 The Purpose of the /static Folder

Flask enforces a specific static/ directory to ensure:

  • Security: Prevents users from accessing arbitrary backend files (like your database or .env).
  • MIME Types: Automatically tells the browser if a file is an image, script, or style.
  • Organization: Provides a predictable URL structure: /static/style.css.

3 Professional Project Structure

Stop putting everything in one file. Real software engineering requires modularity.

project/
├── app/
├── __init__.py # Turns folder into a package
├── routes/ # Handlers
├── models/ # DB Logic
static/ # Global Assets
templates/ # HTML Files
├── migrations/
├── venv/ # (DO NOT GIT COMMIT)
├── requirements.txt
└── run.py

4 Separation of Concerns

Part Responsibility
Routes Handle URLs & Requests
Models Database logic & Data rules
Templates UI & Presentation (HTML)
Static Assets (CSS/JS/Images)
Config Environment Settings & API Keys

5 Rules & Security

Avoid These

  • Inline CSS/JS inside templates
  • Committing venv/ to Git
  • Serving user uploads directly from static/
  • Using random filenames (be descriptive!)

Do These

  • Keep static files in Version Control
  • Structure templates to mirror routes
  • Use external CSS for browser caching
  • Structure early to avoid "technical debt"
