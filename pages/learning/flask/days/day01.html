Day 1

How the Web Actually Works

Before writing Flask code, you must understand the conversation between a browser and a server.

1 The Client-Server Model

The web is simply computers talking to other computers. Your browser (Chrome, Firefox) is the Client, and the computer running your code is the Server.

[Image of client server architecture diagram]

Client

Sends requests. Lives in the browser. Displays the UI (HTML/CSS).

Server (Flask)

Listens for requests. Runs Python logic. Talks to databases.

2 HTTP Methods (The Verbs)

GET

Fetch data. Safe to call multiple times. Example: Loading your profile page.

POST

Create data. Submitting a form or uploading a file. Example: Registering a user.

PUT

Update data. Replaces existing information. Example: Changing your username.

DELETE

Remove data. Deletes a resource. Example: Deleting a post.

3 HTTP Request & Response

// Request Example
GET /profile?id=3 HTTP/1.1
Host: example.com
Content-Type: application/json
// Response Example
HTTP/1.1 200 OK
Content-Type: text/html
Body: <h1>Welcome!</h1>

4 Status Codes (The Cheat Sheet)

2xx (Success)

200 OK, 201 Created

3xx (Redirection)

301 Moved, 302 Found

4xx (Client Error)

400 Bad Request, 404 Not Found

5xx (Server Error)

500 Internal Server Error
