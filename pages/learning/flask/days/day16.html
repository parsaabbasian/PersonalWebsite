Channels in Go

Mastering communication between goroutines. Learn to synchronize execution and share data safely without shared memory.

1 What is a Channel?

A channel is a pipe that goroutines use to communicate. It allows one goroutine to send values to another, ensuring safe data transfer.

‚ÄúDo not communicate by sharing memory; share memory by communicating.‚Äù

Creating a Channel

ch := make(chan int)
// Channels are type-safe

Send & Receive

ch <- 10 // Send
val := <-ch // Receive

2 Blocking Behavior

Critical Concept

By default, sends and receives block until the other side is ready. This allows goroutines to synchronize without explicit locks or condition variables.

ch := make(chan int)
ch <- 5 // ‚ùå Deadlock if no one is receiving!

3 Synchronization Example

Channels provide a clean way to fix the problem where main() exits before goroutines finish.

sync.go
func main() {
    done := make(chan bool)

    go func() {
        fmt.Println("Processing...")
        done <- true // Signal completion
    }()

    <-done // Block here until signal received
}
