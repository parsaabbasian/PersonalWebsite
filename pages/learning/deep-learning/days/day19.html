2 The API Request Lifecycle

When a client calls an API, the Flask process changes slightly. Instead of parsing form fields, it parses JSON bodies; instead of rendering templates, it Serializes data.

Serialization Converting complex DB objects into plain JSON strings.
Validation Checking that incoming JSON has the right structure and types.

3 APIs are Promises

The Contract Mindset

Unlike a website where you can change the UI anytime, an API response is a "Contract." If you change a field name, you break the mobile apps and services that rely on you. This is why Versioning (e.g., /v1/) is mandatory.

4 Status Codes are the UI

In APIs, there are no error pages. The HTTP Status Code is the only way the client knows what happened.

201 Created Success (POST)
422 Unprocessable Validation Error
401 Unauthorized Token Required
403 Forbidden No Permission
