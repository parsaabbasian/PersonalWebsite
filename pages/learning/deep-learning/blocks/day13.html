Day 13

Application Architecture

Scaling Flask from a single script to a professional, maintainable codebase.

1 The Application Factory

Instead of a global app object, we use a function to create the app instance. This allows for easier testing and multiple configurations.

def create_app(config_class=Config):
app = Flask(__name__)
app.config.from_object(config_class)
# Initialize extensions
db.init_app(app)
migrate.init_app(app, db)
return app

2 Blueprints: Modular Routing

Blueprints allow you to organize your application into distinct components, such as auth, main, and api.

Structure

Group related routes, templates, and static files into independent folders.

Reusability

Easily plug features in and out of different projects.

Day 13 Outcome

You are now thinking like a software architect. You should be able to:

  • Convert a single-file app into a Factory pattern
  • Register Blueprints for different modules
  • Organize folder structures for large projects
