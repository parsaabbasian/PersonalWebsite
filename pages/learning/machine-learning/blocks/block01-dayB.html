Block 1 • Day B

Problem Framing & Data

Most projects fail here, not in the code. Learn to ask the right questions and structure your data for success.

1 Features vs Labels

Understanding your inputs (X) and your target (y) is the foundation of any supervised learning project.

Features (X)

The inputs. Information the model looks at to make a decision (e.g., house size, email text).

Labels (y)

The answers. What the model is trying to predict (e.g., house price, spam/not spam).

Mental Rule: Features explain. Labels decide.

2 The Splitting Strategy

Memorization is not learning. We split data to ensure the model can generalize to data it has never seen before.

Training Set (70%) Used to learn rules
Validation Set (15%) Used to tune complexity
Test Set (15%) Final unbiased evaluation

3 Data Leakage: The Silent Killer

What is it?

Data leakage happens when information from the future (or the test set) accidentally leaks into training. It leads to fake high accuracy in development, but total failure in the real world.

Example: Predicting a patient's disease using a feature that was recorded *after* the diagnosis was already made.

⚙️ The ML Mindset

  • Objective: What exactly am I trying to predict? (churn? price? cat?)
  • Data: Do I have historical labels (answers) for these features?
  • Evaluation: How will I know if the model is actually learning or just memorizing?
