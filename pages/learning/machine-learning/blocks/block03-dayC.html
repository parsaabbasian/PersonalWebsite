Day 12

CRUD Patterns

Mastering the fundamental operations of every backend application: Create, Read, Update, and Delete.

1 The CRUD/HTTP Mapping

CRUD is a design pattern that describes how data flows between the user and the database. Professionally, we map these actions to specific HTTP Methods.

Action HTTP Flask Logic
Create POST Add new resource
Read GET Fetch resource(s)
Update PUT/PATCH Modify existing data
Delete DELETE Remove resource

2 Creation & Validation Lifecycle

Validate First

Creation or updates should never happen without re-validating the data on the server.

Commit Transaction

Use SQLAlchemy sessions to ensure the data is written permanently.

The Update Rule

Never assume existing data is correct. When updating, ensure you aren't accidentally overwriting fields the user didn't intend to change.

3 Delete: The Dangerous Operation

Hard Delete

Permanently removes the row from the database. Irreversible and can break relationships (e.g., deleting a user but leaving their posts).

Soft Delete (Professional Choice)

Instead of deleting, we set a is_deleted flag to True. The data stays in the DB but is hidden from the UI. Safer and allows for "Undo" functionality.

4 Pagination & Security

Pagination

Never load an entire table. As your app grows, fetching 10,000 rows will crash your server. Design for small chunks (e.g., 20 items at a time).

Ownership

CRUD is a security boundary. Always check: "Does this user own the resource they are trying to Update or Delete?"

Day 12 Outcome

You have mastered the data-flow patterns used by professional backends. You should now be able to:

  • Map CRUD to HTTP Methods
  • Explain why Delete is a business decision
  • Design routes for single vs multiple resources
  • Understand the need for Pagination
