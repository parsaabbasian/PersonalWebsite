Control Flow: Logic & Safety

Mastering match, loops, and why Rust's control flow is fundamentally safer than other languages.

1 if is an Expression

In Rust, if isn't just a statement; it's an expression that returns a value. This allows for cleaner variable assignments based on conditions.

let y = if x > 5 { 1 } else { 0 };
Python: ❌
C++: ❌
Go: ❌
Rust: ✅

2 Safe Looping Patterns

Rust provides three main ways to loop. The for loop is preferred because it uses iterators, eliminating out-of-bounds risks.

loop

Infinite until break. Can return values.

while

Conditional loop. Standard behavior.

for

Safe iterators. Use 1..4 for ranges.

3 match: Rust’s Superpower

Matches in Rust are exhaustive. The compiler will not allow you to forget a case, effectively turning logic errors into compile-time errors.

match number {
    1 => println!("One"),
    2 => println!("Two"),
    _ => println!("Exhaustive catch-all"),
}

Logic Protection

Rust prevents real-world bugs common in other systems languages:

  • C++ Risk Accidental switch fall-through.
  • Rust Fix Match arms never fall through; they are independent.
  • Benefit Enums + Match = Automatic compile-time documentation.
