Day 10

SQLAlchemy ORM

Bridging the gap between Python Objects and Relational Databases.

1 The Core Problem: Language Mismatch

Databases speak SQL (Relational). Python speaks Objects. Without a bridge, you have to write messy SQL strings inside your Python code.

Manual Way (Hard)

"INSERT INTO users (name) VALUES ('" + name + "')"

ORM Way (Smart)

user = User(name="Alice")

2 Mapping Tables to Classes

Database (SQL) Python (SQLAlchemy)
Table Class
Row Object Instance
Column Attribute
Primary Key Object ID

3 The Session: The Gatekeeper

SQLAlchemy doesn't save data immediately. It uses a Session to track changes. Think of it as a "drafting area."

1

Add to Session

"I want to save this object later."

2

Commit Transaction

"Write all my changes to the disk now."

Backend Rule: Nothing is saved automatically. If you don't commit, your data is lost when the request ends.

4 Thinking in Objects

Queries

Instead of SQL SELECT, you ask the Class:
"User, give me the person with ID 5."

Relationships

Instead of manual joins, you access attributes:
"user.posts" gives you their articles automatically.
