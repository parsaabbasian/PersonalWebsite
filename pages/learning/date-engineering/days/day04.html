Day 4

Functions, Expressions & Return Values

Mastering the building blocks of Rust logic and avoiding the "Semicolon Trap."

1 Defining Functions

In Rust, function signatures must be explicit. There is no type-guessing for parameters or return values.

fn add(a: i32, b: i32) -> i32 {
    a + b // Implicit return (no semicolon!)
}
  • Parameters must have explicit types.
  • Return type is marked with ->.

2 Expressions vs Statements

This is the most critical concept for Day 4. Expressions evaluate to a value, while Statements perform an action and return nothing (()).

Expression

5 + 3

Produces 8. No semicolon.

Statement

let x = 5;

Assigns value. Has semicolon.

3 The Semicolon Trap

Adding a semicolon to the last line of a function changes its return value to Unit (). This is the #1 cause of "Mismatched Types" errors for beginners.

// ERROR: Expected i32
fn x() -> i32 {
  5;
}
// CORRECT
fn x() -> i32 {
  5
}

4 Scope & Cleanups

Block Scoping

Rust automatically destroys values when they go out of scope. This is why we don't need a garbage collector.

{
  let x = 10; // x is born
} // x is dropped from memory
  • Predictable You know exactly when memory is freed.
  • Efficient No "Garbage Collection pauses" like Python or Go.
