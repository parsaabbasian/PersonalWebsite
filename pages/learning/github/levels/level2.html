Level 2

Branching & Merging

Mastering parallel universes. Experiment safely without breaking your main code.

1 The Concept

Imagine you are building a house (your code). You want to see if a new balcony looks good, but you don't want to actually smash a hole in your perfectly good wall just to test it.

In Git, you create a Branch. This creates a "Parallel Universe" copy of your code.

Main Branch

The "Real World" (Keep this safe and working).

Feature Branch

The "Experiment" (Do whatever you want here; if it breaks, who cares?).

2 Creating a Branch (The Experiment)

Let's add a blue button to your website, but do it safely on a separate branch.

  1. Create and Switch to a new branch:

    We will use the modern command switch -c (Create and Switch).

    git switch -c feature-blue-button

    Your terminal should now say you are on branch 'feature-blue-button'.

  2. Modify your code:

    Open your index.html file. It should look just like it did before. Add a button code:

    <h1>Hello World</h1>
    <p>This is my first Git project!</p>
    <button style="background-color: blue; color: white;">I am a Blue Button</button>
  3. Save (Commit) to this branch:
    git add index.html
    git commit -m "Added a blue button"

3 The "Magic" Switch

This is the moment that confuses beginners until they see it happen.

1. Switch back to Main:

Go back to your original timeline.

git switch main

2. Look at your file:

Open index.html (or look at it in your browser). The button is gone!
Git instantly reverted your files to how they look in the main universe. Your work isn't lost; it's just safely stored in the other branch.

4 Merging (Bringing it Home)

[Image of git merge diagram]

You decided the blue button looks great. Let's bring that change into the main "Real World."

  1. Stay on Main: (Ensure you are on the branch you want to receive the changes).
    git status

    Should say "On branch main"

  2. Merge: (Absorb the history of the other branch).
    git merge feature-blue-button

    Open your index.html file now. The button has magically appeared in Main!

5 Ignoring Junk (.gitignore)

Sometimes you have files you never want to save to Git (e.g., hidden system files, passwords, or huge video files).

  1. Create a file named .gitignore:

    Create this file in your main folder.

  2. Add rules to it:

    Open .gitignore in your editor and type the names of files to ignore. Let's pretend we have a secret file.

    secret-passwords.txt
    *.log
  3. Test it:

    Create a file called secret-passwords.txt. Now run:

    git status

    Git will NOT list secret-passwords.txt as a new file. It is invisible to Git.

6 The Emergency Pause Button (Stash)

[Image of git stash diagram]

Imagine you are working on a "Red Button" branch. You have half-finished code that is broken. Suddenly, your boss says, "Fix a typo on Main right now!"

You can't switch branches because your current code is messy. You need to "Stash" it (put it in a drawer).

1. Stash your changes

git stash

Your folder is now clean (reverted to the last commit), and your messy work is saved in a temporary memory slot.

2. Do your other work...

3. Bring the work back

git stash pop

Your messy work reappears exactly as you left it.

Mastery Notes: Cheat Sheet

1. Branching (Parallel Universes)

  • git switch -c <name> // Create & Switch
  • git switch main // Return to original code
  • git branch // See all branches

2. Merging (Combining)

  • git switch main // 1. Go to destination
  • git merge <name> // 2. Absorb changes

3. Ignoring Files

  • Create file .gitignore
  • List file names or patterns (like *.log) inside.
  • Purpose: Keeps the repo clean and secure.

4. Stashing

  • git stash // Save messy work
  • git stash pop // Restore work
