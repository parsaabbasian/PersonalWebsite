Level 1

The Foundation

Understanding the core difference between Git and GitHub, and setting up your first repository.

1 The Big Picture

Git

A "Time Machine" on your computer. It saves versions of your files locally. You don't need internet to use it.

GitHub

A "Cloud Storage" (like Google Drive) for your Git history. It lets you share your work with others online.

2 First-Time Setup

Before you can save anything, you must introduce yourself to Git. Open your terminal and run these once:

git config --global user.name "Your Name"

git config --global user.email "your.email@example.com"

3 Starting a Local Web Project

  1. Create a folder:
    mkdir MyWebProject
    cd MyWebProject
  2. Initialize Git: (Tells Git to watch this folder)
    git init

    You will see "Initialized empty Git repository".

  3. Create your HTML file:
    <h1>Hello World</h1>
    <p>This is my first Git project!</p>

4 The Saving Workflow (Loop)

1. Check Status

See what Git sees. New files will be red.

git status

2. Stage (Add)

Move file to the "Waiting Room".

git add index.html

3. Commit (Save)

Take a permanent snapshot.

git commit -m "Created homepage"

5 Connecting to GitHub

  1. Go to github.com and create a New Repository named MyWebProject.
  2. Important: Keep "Add a README" unchecked.
  3. Copy the URL (e.g., https://github.com/User/Repo.git).
  4. Link your terminal:
    git remote add origin <PASTE-URL-HERE>
  5. Rename Branch & Push:
    git branch -M main
    git push -u origin main

    The -u flag stands for "Upstream" (technically --set-upstream). It links your local branch to the GitHub one so you can just type git push later.

Mastery Notes: Cheat Sheet

1. Setup (One time only)

  • git config --global user.name "My Name"
  • git config --global user.email "me@example.com"

2. The Loop

  • git status // Check changes
  • git add . // Stage ALL files
  • git commit -m "Message" // Save snapshot

3. Uploading

  • git push // Uploads code (after first setup)
