Level 4

The Time Lord

Rewriting history, fixing mistakes cleanly, and recovering lost data.

1 Rebase vs. Merge

Merge

Tie two branches together with a knot. It preserves history exactly as it happened (even the messy parts).

Analogy: A truthful diary.

Rebase

Rewrite history to make it look like a straight line. It moves your work to the "tip" of the main branch.

Analogy: A polished resume.

Warning: Never use rebase on a shared branch (like main). Only use it on private feature branches.

2 The Project (Squashing Commits)

You made 5 messy commits and want to combine them into one perfect commit.

1. Start interactive rebase

git rebase -i HEAD~5

2. The Editor Opens (Change 'pick' to 'squash')

pick a1b2c Added login
squash d4e5f Fix typo
squash g7h8i Forgot semicolon
squash j0k1l Wip
squash m3n4o Final fix

3. Result: Git combines them. You type a new message like "Feature: Login System". History is now clean!

3 Reset vs. Revert

1. Git Reset (Time Travel)

"It never happened." You jump back in time.

git reset --soft HEAD~1 // Keep files changed
git reset --hard HEAD~1 // Delete changes (Dangerous)

2. Git Revert (The Apology)

"It happened, but I am fixing it." Safe for shared branches.

git revert <commit-hash>

4 The "Aha!" Moment (Reflog)

You accidentally ran reset --hard and deleted your work. It is not gone. Git keeps a secret diary called Reflog.

// 1. View the secret diary
git reflog

a1b2c HEAD@{0}: reset: moving to HEAD~1

d4e5f HEAD@{1}: commit: My important work

// 2. Rescue the lost commit
git reset --hard d4e5f

Poof! Your deleted files are back.

5 Cherry-Picking

Sometimes you don't want to merge a whole branch. You just want one specific commit.

git cherry-pick <commit-hash>

Git copies that specific commit and pastes it onto your current branch.

Level 4 Cheat Sheet

1. Clean History (Squash)

git rebase -i HEAD~n

2. Undo Commands

git reset --soft HEAD~1 // Undo commit only
git reset --hard HEAD~1 // DELETE code
git revert <hash> // Safe undo

3. The Savior (Reflog)

git reflog // View history
git reset --hard <hash> // Recover

4. Cherry Pick

git cherry-pick <hash> // Copy commit
