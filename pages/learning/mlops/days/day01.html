Day 1

Environment Setup & First Program

Moving from the "Why" to the "How." Today we build your foundation and run your first binary.

1 The Rust Mindset

Rust pushes correctness to compile time. It solves the specific problem of writing fast, low-level code without the typical memory bugs found in C++ or the performance trade-offs of Python.

Python Mindset

"If it runs, we'll see if it breaks."

Rust Mindset

"If it compiles, it is likely correct."

2 Installing the Toolchain

We use rustup to manage the compiler (rustc) and package manager (cargo).

# Install command for Unix-based systems
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
rustc --version
cargo --version

3 Creating Your First Project

Rust expects a structured project layout by default. Use Cargo to scaffold it instantly.

// Create and enter project
cargo new hello_rust
cd hello_rust
// Resulting Structure
hello_rust/
├── Cargo.toml (Metadata/Dependencies)
└── src/
└── main.rs (EntryPoint)

4 The First Code: src/main.rs 

fn main() {
    println!("Hello, world!");
}

Key takeaway:

The ! in println! means it is a Macro. Code generation happens here to ensure your formatting is checked at compile-time.

5 The "Implicit" Test

Unlike Python, Rust allows nothing to exist implicitly. Variables must be declared.

WON'T COMPILE 
x = 5;
RUST WAY 
let x = 5;
