Day 2

Variables, Mutability & Types

Understanding how Rust's type system and strict variable rules prevent bugs before they happen.

1 Immutability & mut

In Rust, variables are immutable by default. If you want to change a value, you must be explicit.

// Immutable
let x = 10;

// Mutable
let mut y = 20;
y = 30;

2 Shadowing vs Mutation

Shadowing allows you to re-declare a variable with the same name, potentially changing its type while keeping the data immutable.

Feature Shadowing Mutation
New Variable? Yes No
Change Type? Yes No
Day 3

Control Flow & Pattern Matching

Mastering match and the power of expressions over statements.

1 The match Superpower

Rust's match is exhaustive. The compiler ensures you have handled every possible case, preventing "unhandled state" bugs common in other languages.

match number {
    1 => println!("One"),
    2 => println!("Two"),
    _ => println!("Anything else"),
}

2 Loops: Safe & Iterative

Rust prefers for loops over iterators to avoid out-of-bounds errors.

Infinite Loop

loop { ... }

Can return values using break.

For Iterator

for item in collection { ... }

No manual index management.
