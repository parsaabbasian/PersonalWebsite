Day 12

Error Handling in Go

Understanding the Go philosophy: errors are values, not exceptions. Master the patterns that keep Go code predictable and explicit.

1 Errors are Values

In Go, error is an interface, not an exception. It is returned like any other value, making flow control explicit.

The Classic Pattern

value, err := someFunction()
if err != nil {
    // handle error
    return
}

‚≠ê You will see this pattern in almost every Go project.

2 Creating & Returning Errors

Simple Errors

Use the errors package for static messages.

import "errors"

errors.New("invalid input")

Formatted Errors

Use fmt.Errorf to include variables in your message.

import "fmt"

fmt.Errorf("invalid age: %d", age)

3 Understanding "nil"

In Go, nil represents "no value". It is the default value for interfaces and errors.

var err error
fmt.Println(err == nil) // true
