Day 26

Go Best Practices

Writing "idiomatic" Go. Learn the conventions and philosophy that separate a Go developer from someone just writing C++ or Java in Go.

1 The Zen of Go

Go values simplicity and readability over "clever" code. If a junior developer can't understand your logic at first glance, it likely isn't idiomatic Go.

  • Explicit over implicit
  • Avoid deep nesting

2 Standard Layout

For scalable applications, follow the community-accepted folder structure to separate entry points from internal logic.

/cmd // Main applications (binaries)
/internal // Private code; cannot be imported by others
/pkg // Reusable library code
/api // API protocol definitions (Swagger, etc.)

3 The Mandatory Toolkit

gofmt

Go code has one standard format. Run gofmt -w . to auto-format everything. There are no arguments about curly brace placement in Go.

go vet

Analyzes source code and reports suspicious constructs, such as printf calls whose arguments do not align with the format string.
