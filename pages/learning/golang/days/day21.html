2 Mini Project: Parallel Data Fetcher

Build a simulation of fetching data from multiple sources (like APIs or Databases) simultaneously using Goroutines and Channels.

fetcher.go
func fetchData(id int, ch chan string, wg *sync.WaitGroup) {
    defer wg.Done()
    time.Sleep(time.Second) // Simulate network delay
    ch <- fmt.Sprintf("Data from Source %d", id)
}

func main() {
    results := make(chan string, 3)
    var wg sync.WaitGroup

    for i := 1; i <= 3; i++ {
        wg.Add(1)
        go fetchData(i, results, &wg)
    }

    wg.Wait()
    close(results)

    for res := range results { fmt.Println(res) }
}

Week 3 Graduation

You have mastered the hardest part of Go: Concurrency. You now understand how to build software that is fast, safe, and efficient.

Review: Re-visit the select statement and ensure you understand how it handles timeouts.

Think: Why did we use close(results) in the project code? (Answer: To signal the range loop that no more data will be sent, allowing it to finish gracefully.)
