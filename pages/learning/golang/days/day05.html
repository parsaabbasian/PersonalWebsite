Day 5

Functions in Go

Mastering modularity in Go: parameters, return values, and the superpower of multiple returns.

1 Syntax & Parameters

Basic Function

A simple function without parameters or returns.

func greet() {
    fmt.Println("Hello from Go!")
}

Passing Parameters

Defining specific types for input values.

func greet(name string) {
    fmt.Println("Hello", name)
}

2 The Go Superpower: Multiple Returns

Go can return more than one value, commonly used for returning both a result and an error status.

func divide(a, b int) (int, bool) {
    if b == 0 {
        return 0, false
    }
    return a / b, true
}

Note: This pattern effectively replaces traditional try/catch exceptions found in other languages.

3 Named Returns & Scope

Named Return Values

Cleanly define return variable names in the signature.

func rect(w, h int) (area int) {
    area = w * h
    return
}

Function Scope

Variables inside functions are local.

  • Exist only inside the braces
  • Do not affect global state directly
