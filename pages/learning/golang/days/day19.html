Day 19

Defer, Panic & Recover

Mastering resource cleanup and Go's unique mechanism for handling unrecoverable errors and program crashes.

1 The defer Keyword

defer schedules a function to run immediately after the surrounding function returns. It is commonly used for resource cleanup like closing files.

LIFO Execution

Deferred functions are pushed onto a stack; the last one deferred runs first.

defer fmt.Println("1")
defer fmt.Println("2")
// Output: 2 then 1

Resource Cleanup

Ensures files or locks are released even if an error occurs.

file, _ := os.Open("t.txt")
defer file.Close()

2 Panic & Recover

What is panic?

A panic immediately stops execution and crashes the program unless intercepted. Use it only for unrecoverable bugs, never for normal error handling.

panic("unrecoverable state")

What is recover?

recover can stop a panic from crashing the program, but it only works inside a deferred function.

defer func() {
    if r := recover(); r != nil {
        fmt.Println("Recovered from:", r)
    }
}()

3 panic vs. error

Situation Recommended Tool
Invalid user input error
File not found error
Nil pointer dereference (Bug) panic
Out of bounds array access panic
