Day 8

Arrays & Slices

Understanding the core difference between fixed-size arrays and dynamic slices, and mastering the append function.

1 The Big Picture: Arrays vs. Slices

Arrays (Fixed)

Size is part of the type. They cannot grow or shrink once declared.

var nums [3]int
nums = [3]int{10, 20, 30}
Slices (Dynamic)

Flexible size. These are used in 99% of Go code because they can grow.

nums := []int{10, 20, 30}
nums = append(nums, 40)

2 Slicing Logic

Use the [start:end] syntax to create a sub-slice. Remember: the end index is NOT included.

nums := []int{10, 20, 30, 40, 50}
// Result: [20 30 40]
fmt.Println(nums[1:4])
// Result: [10 20 30] (from start)
fmt.Println(nums[:3])

3 Length & Capacity

Length (len)

The number of elements currently in the slice.

Capacity (cap)

The total space allocated in the underlying array.

4 Looping over Slices

The range keyword is the idiomatic way to loop through slices in Go.

for index, value := range nums {
    fmt.Println(index, value)
}

// If you don't need the index, use the blank identifier:
for _, value := range nums { ... }
