Environment & Config

Decoupling code from configuration. Learn how to manage secrets and settings safely across different environments.

1 Why Environment Variables?

Environment variables are key‚Äìvalue pairs provided by the OS. They are critical for keeping Secrets (API keys) and Configuration (Database hosts) out of your source code.

‚ùå Bad Practice

const apiKey = "SECRET_123"

‚úÖ Good Practice

2 Implementation in Go

Go's os package makes it simple. If a variable doesn't exist, it returns an empty string.

// Get value
port := os.Getenv("PORT")
// Handle default
if port == "" {
    port = "8080"
}

3 The .env File Pattern

Local Development

In real-world projects, we use a .env file. Never commit this to Git.

PORT=8080
DB_NAME=students_db
DEBUG=true
