Goroutines: Concurrency in Go

Introduction to Go's lightweight threading model and the fundamental difference between concurrency and parallelism.

1 Concurrency vs Parallelism

Concurrency

Dealing with many tasks at once. It is about structure and managing multiple execution paths.

Parallelism

Doing many tasks at the same time. It is about execution on multiple CPU cores.

‚ÄúConcurrency is about structure, parallelism is about execution.‚Äù

2 What Is a Goroutine?

A goroutine is a lightweight thread managed by the Go runtime. They are extremely "cheap" to create, allowing you to run thousands simultaneously using the go keyword.

example.go
func sayHello() {
    fmt.Println("Hello from goroutine")
}

func main() {
    go sayHello() // Starts concurrently
    fmt.Println("Hello from main")
}

Note: Without synchronization (like time.Sleep or WaitGroups), the program might exit before the goroutine finishes its work.

3 Advanced Usage

Anonymous Goroutines

go func() {
    fmt.Println("Async task")
}()

Race Conditions

Sharing memory between goroutines without protection is dangerous.

count := 0
go func() { count++ }()
